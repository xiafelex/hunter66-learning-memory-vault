# 设计说明

## 参考思路

这个仓库借鉴了三类成熟思路：

- 间隔复习：错题和知识点需要按时间重复遇见，而不是当天订正后就消失。
- 知识追踪：同一个孩子对不同知识点的掌握度不同，记录应围绕“知识点-证据-掌握度”展开。
- 错因分析：错题价值不在于保存题目，而在于识别稳定错因，例如审题、概念、计算、表达、记忆、习惯。

## 为什么用 Markdown 和 Git

- Markdown 方便人直接读，也方便 AI 读取。
- Git 能保留学习变化历史，适合长期沉淀。
- 文件夹结构比单一大表更适合保存老师要求、错题说明和阶段复盘。

## 不建议一开始做得太复杂

小学二年级阶段，最重要的是形成稳定记录节奏：

- 每周 10-20 分钟整理一次。
- 每次只记录能指导下一次学习的内容。
- 对反复错误建立薄弱点，而不是把每道题都长期追踪。

## 隐私边界

本仓库可能涉及未成年人学习材料。建议：

- 公开仓库中不放孩子照片、班级群截图、学校全称、老师完整姓名。
- 原始图片如果必须保存，优先放在本地私有目录或私有仓库。
- Markdown 记录中使用摘要，不复制敏感聊天全文。
