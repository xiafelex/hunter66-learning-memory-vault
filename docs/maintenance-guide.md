# 维护指南

## 每日或随手记录

适合记录：

- 老师新要求。
- 当天出现且值得复盘的错题。
- 孩子明显卡住的知识点。
- 家长陪练时发现的有效讲法。

不必记录：

- 已经稳定掌握的普通练习题。
- 没有复盘价值的一次性小失误。
- 无法回溯来源的模糊印象。

## 每周复盘

每周建议检查：

- 本周新增错题数量。
- 出现 2 次以上的错因。
- 仍有效的老师要求。
- 下周只保留 1-3 个最重要的训练重点。

可以运行：

```bash
python3 scripts/build_weekly_review.py
```

脚本会读取 `data/` 下的 Markdown front matter，生成一个基础周复盘草稿。

## 错题记录要点

一条好错题至少包含：

- 题目来源和日期。
- 错误答案或错误做法。
- 正确答案和关键步骤。
- 错因分类。
- 给六六看的下次提醒。
- 下次复习日期。

## 薄弱点升级规则

满足任一条件，可从错题升级为薄弱点：

- 同一错因在两周内出现 2 次以上。
- 老师明确提醒需要加强。
- 考试或单元练习中集中失分。
- 孩子会做但稳定性差，影响作业效率。

薄弱点达到以下状态可降级：

- 连续两周相关题目正确率稳定。
- 孩子能独立说出方法和检查点。
- 新情境题目也能迁移。
